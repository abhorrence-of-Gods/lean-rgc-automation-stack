# S6 Pre-Registration: lemma foundry v0 (analogy-guided asset casting)

Status: registered before execution. Thresholds frozen; amendments
require a new dated section with a reason.

## Question

Can LLM-conjectured, verifier-cast lemmas move the accumulation organ off
zero — i.e., produce NON-TRIVIAL inter-theorem value where M1 measured
~0% at the text gauge? This is the go/no-go for the (b)-organ
(structural distillation into the lemma spanner).

## Frontier and proposal prior

- Frontier inventory F: the M1 unique K1 residual-goal keys extracted
  from PILOT-ERA partial rows only (341 keys; a stratified sample of up
  to 100 keys is the v0 proposal set).
- Proposal prior U0: the pod LLM conditioned on RAW context (statement +
  residual goal text + last error), per the E1 lesson. k=4 candidates
  per key: each candidate = a self-contained lemma statement plus a
  proof script.
- Casting chi(b): the lemma theorem (statement := by script) must verify
  through the bulk lane. v0 DISCLOSURE: minimal-support canonicalization
  (M2) is not yet available; cast lemmas are recorded as-stated and the
  exact-support rung is deferred to S6-full after S2 lands.

## Frozen anti-Goodhart clauses

1. SPATIAL split: Delta is never measured on the goals used to propose.
2. TEMPORAL split (walk-forward): proposals are conditioned ONLY on
   pilot-era rows (2026-07-03/04); value is measured ONLY on G1-era
   residual goals (2026-07-05 wave rows). Static held-out splits do not
   cover distribution shift induced by the foundry itself; the pilot->G1
   boundary is the natural first walk-forward step.
3. Triviality stratification with MANDATORY manual inspection of every
   claimed non-trivial hit (M1 lesson: the automated trivial detector
   has false negatives — forall-wrapped True, bare-mvar targets).
4. Net pollution: C_maintain is measured as retrieval degradation —
   premise-index size and fixed-query retrieval precision before/after
   adding cast lemmas — not storage.
5. Double ledger: casting failures are logged with candidate features to
   the foundry's soft table (U calibration only); they never gate.
6. Recursion truncation: the asset-level twist (candidate features ->
   cast probability x value) is LIMITED TO ONE LEVEL (log-linear), and a
   second level may only be proposed after a measured calibration-error
   economics case; none is built in S6.
7. Greedy accounting: v0 runs gamma=0 (myopic Delta). Known bias
   disclosed: scaffolding lemmas whose value is only realized through
   later assets are systematically undervalued. The (1-1/e)-type greedy
   guarantee applies only under monotone submodular value with uniform
   costs; with non-uniform C_maintain a cost-ratio greedy variant is
   required — v0 reports raw Delta and cost separately rather than
   claiming the guarantee.

## Frozen measurables and gates

- A cast rate: p_cast = fraction of proposed candidates that verify.
- B post-cast value, two forms on the WALK-FORWARD set (G1-era residual
  goals): (i) spanner-hit: a cast lemma's statement matches (K1/K3
  gauge) a held-out residual goal; (ii) closure: `exact/apply <lemma>`
  audited on held-out goal-only-reconstructible residual goals closes
  or advances them (verifier-measured Delta proxy).
- C net value I(U) = casts x mean value - sampling/verify cost
  (reported in verifier calls and LLM calls).

GO (foundry lives): p_cast >= 0.10 AND >= 5 distinct held-out
non-trivial hits/closures AND manual inspection confirms all of them.
DEGENERATE: otherwise — accumulation remains offline (M1's verdict
extends to the lemma gauge for this generator), and the foundry is
shelved until S2 (minimal support) or a stronger proposal prior changes
an input.

## Threats acknowledged in advance

- The generator is the same 7B family that produced the residual goals;
  self-affinity may inflate hit rates relative to independent targets.
- Lemma statements are accepted as-stated (no generality canonicalization
  in v0): measured value is a LOWER bound on the canonicalized value but
  an UPPER bound may not transfer across libraries.
- The closure proxy (exact/apply) misses lemmas useful only through
  rewriting or automation; report is per-proxy, not a total value claim.
